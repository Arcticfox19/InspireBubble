# 测试指南

## 启动开发服务器

```bash
npm run dev
```

开发服务器将在 `http://localhost:5173` 启动（默认端口）。

## 测试步骤

### 1. 配置 API Key

在应用界面的"API Key 设置"部分：
- 输入您的 Gemini API Key
- 点击"保存"按钮
- 或者使用环境变量（在 `.env` 文件中设置 `VITE_GEMINI_API_KEY`）

**获取 Gemini API Key：**
1. 访问 [Google AI Studio](https://makersuite.google.com/app/apikey)
2. 登录您的 Google 账户
3. 创建新的 API Key
4. 复制并粘贴到应用中

### 2. 输入内容

在"输入内容"区域：
- 粘贴 AI 对话记录
- 或输入任意文本内容
- 支持 Markdown 格式

**测试示例内容：**

```
React 是一个用于构建用户界面的 JavaScript 库。它采用组件化开发方式，让代码更易维护和复用。使用 React Hooks 可以更简洁地管理组件状态。JSX 语法让编写 UI 代码更加直观。
```

### 3. 转换内容

1. 点击"转换为对话格式"按钮
2. 等待 AI 处理（会显示加载动画）
3. 查看转换结果

### 4. 查看预览

转换成功后，右侧预览区域会显示：
- 标题（AI 生成的小红书风格标题）
- 对话内容（用户问题和 AI 回答）

## 常见问题

### Q: API Key 错误
**A:** 请检查：
1. API Key 是否正确
2. 是否已启用 Gemini API
3. 是否有足够的配额

### Q: 转换失败
**A:** 可能的原因：
1. 输入内容为空
2. API Key 未设置
3. 网络连接问题
4. API 配额已用完

### Q: 开发服务器无法启动
**A:** 检查：
1. Node.js 版本（推荐 18+）
2. 依赖是否已安装（运行 `npm install`）
3. 端口 5173 是否被占用
